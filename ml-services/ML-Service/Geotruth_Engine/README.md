## рҹҢ§ Why GeoTruth Exists

A food delivery partner in Chennai earns вӮ№800вҖ“вӮ№1,200 per day.  
When a cyclone, flash flood, or curfew hits their zone вҖ” they earn вӮ№0.  
Parametric insurance was supposed to fix this.  
**GPS spoofing broke it.**

A syndicate of 500 delivery workers organized via Telegram used GPS spoofing apps to fake their locations inside red-alert weather zones вҖ” draining liquidity pools and destroying trust in the entire model.

**GeoTruth is the answer.**

Not a patch. Not a GPS upgrade. A complete rearchitecting of what *presence verification* means in the age of software-defined location.

> *"GPS tells you coordinates. GeoTruth tells you whether those coordinates are lived reality or a digital lie."*

### Minimal вҖ” Verify a claim in 5 lines

```python
from geotruth import GeoTruthEngine
from geotruth.core.schemas import ClaimVector

engine = GeoTruthEngine()

claim = ClaimVector(

    timestamp=1711900000,

    variance_1h_to_30m_ago=0.82,
    variance_last_30m=0.04,
)

result = engine.verify(claim)

print(result.coherence_score)      # 0вҖ“100
```

### Score raw inertial data (no full ClaimVector needed)

```python
from geotruth.layers.inertial import score_raw

# Genuine worker: was active, took shelter

print(result.confidence)    # 0.94

    variance_prior=0.05,
    variance_current=0.03,
)
```

## рҹ“Ӣ ClaimVector Schema

The `ClaimVector` is the JSON object a worker's device sends to the GeoTruth MCP server. All fields beyond `worker_id`, `claimed_pincode`, and `timestamp` are optional вҖ” GeoTruth scores with whatever data is available.

```python
class ClaimVector(BaseModel):
    # в”Җв”Җ Required в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    worker_id:              str           # Anonymized worker ID
    claimed_pincode:        str           # 6-digit Indian pincode
    timestamp:              int           # Unix epoch at claim initiation

    # в”Җв”Җ L1 Barometric в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    device_barometer_hpa:   Optional[float]   # Raw reading from phone sensor
    # None вҶ’ AnchorDeviceProxy used, weight reduced, grace_flag=True

    # в”Җв”Җ L2 Acoustic в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    acoustic_feature_vector: Optional[List[float]]  # 14-float YAMNet output
    mic_permission_granted:  Optional[bool]          # False вҶ’ layer excluded
    # Raw audio NEVER transmitted вҖ” only the 14-float vector

    # в”Җв”Җ L3 Network в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    cell_tower_hashes:      Optional[List[str]]  # SHA-256 of MCC+MNC+LAC+CID
    wifi_ssid_hashes:       Optional[List[str]]  # SHA-256 of SSIDs visible

    is_mock_location:       Optional[bool]       # Android isFromMockProvider
    # Raw SSIDs and Cell IDs NEVER transmitted вҖ” SHA-256 hashes only

    # в”Җв”Җ L4 Inertial в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    variance_1h_to_30m_ago: Optional[float]  # Accelerometer variance, prior window
    variance_last_30m:      Optional[float]  # Accelerometer variance, current window
    gyroscope_available:    Optional[bool]   # False вҶ’ reduced weight + grace_flag

    # в”Җв”Җ L5 Zone (server-resolved вҖ” no device input needed) в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    # GeoTruth fetches zone order volume from platform API automatically

    # в”Җв”Җ L6 Behavioral (server-resolved) в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
    # GeoTruth queries worker profile from PostgreSQL automatically

    # в”Җв”Җ Offline Buffer Flag в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
```

### The 5 Training Archetypes

```
ARCHETYPE 1 вҖ” Genuine Stranded (20% of dataset)
  Prior variance: 0.55вҖ“1.0  (active delivery)
  Current variance: 0.01вҖ“0.12  (took shelter)
  Label: 1 (genuine)

ARCHETYPE 2 вҖ” Genuine Slow Stop (20%)
  Prior variance: 0.30вҖ“0.65  (moderate activity)
  Current variance: 0.05вҖ“0.20  (gradual slowdown)
  Label: 1 (genuine)

ARCHETYPE 3 вҖ” Fraud: Stayed Home (25%)
  Prior variance: 0.00вҖ“0.12  (sedentary)
  Current variance: 0.00вҖ“0.08  (still sedentary)
  Label: 0 (fraud)

ARCHETYPE 4 вҖ” Fraud: Spoof Activation (20%)
  Prior variance: 0.02вҖ“0.15  (sitting at home)
  Current variance: 0.10вҖ“0.35  (picked up phone)
  Label: 0 (fraud вҖ” reverse delta)

ARCHETYPE 5 вҖ” Fraud: Driving & Claiming (15%)
  Both windows: 0.30вҖ“0.80  (near-zero delta)
  Label: 0 (fraud вҖ” no disruption event)
```
